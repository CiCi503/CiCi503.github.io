Hexo + GitHub搭建个人博客(进阶篇)

发表于 2016-12-08 | 分类于 Hexo | | 阅读量

根据Hexo + GitHub搭建个人博客(基础篇)的教程，已经可以搭好一个博客了，这篇文章主要讲述如何对博客进行个性化定制。

博主使用的是Next主题，Next官方文档已经给出了比较全面的教程，这篇文章主要是讲述一下本博客进行的一些个性化设置。

先澄清两个概念，站点配置文件和主题配置文件。站点配置文件是在你的Blog最外层目录下的_config.yml文件，主要是对站点的基本设置；主题配置文件是在Blog->themes->next->_config.yml文件，是对此主题进行的设置。

  3. 创建站点，填写站点相关信息。 多说域名 这一栏填写的即是你的 duoshuo_shortname，如图：
  4. 创建站点完成后在站点配置文件，即中新增 duoshuo_shortname 字段，值设置成上一步中的值。

设置评论栏的css

进入多说后台：后台管理->自己的站点名称（我的是CiCi’s Blog）->设置->自定义css，在里面贴上自己评论栏的css代码就可以了。将我的代码分享给大家~

多说头像链接

发表评论后，点击自己的头像，希望返回自己的博客主页，博主是用微博账号注册的多说，默认就返回博主的微博主页了，那怎样设置默认返回链接呢？方法如下：

  1. 进入多说后台，右上角点击自己的账号->个人资料
  2. 填写邮箱
  3. 选择自定义，并将链接输入到自定义中。

    Attention: 这里一定要填写邮箱，要不然自定义里面的链接是保存不了的，一直提示‘服务器错误，请联系客服人员’

使用本地图片

问题描述

如何在Hexo中使用本地图片呢？很多时候常常在本地编辑器中可以正确找到图片并实时预览了，可是在博客里总是加载不了，这是怎么回事呢？

问题解释

hexo g后，会生成一个public文件夹，这个public文件夹就是html所在的地方。所以，我们要看这个html文件和图片的相对位置关系，而不是本地_posts目录下.md文件和图片的相对位置关系。

解决方法

在_posts中，每新建一个.md文件的同时就新建一个同名的文件夹，并将图片放到同名文件夹中。
以test.md为例，比如要在test.md中引用1.jpg的图片，那么只需在test.md中添加

有一个插件自动帮我们实现了自动新建同名文件夹的功能CodeFalling/hexo-asset-image
在博客目录下执行

这个小程序帮我们在新建一个.md文档的同时，生成一个同名文件夹。我们只需要把这个.md文档中需要用到的图片存储在同名文件夹中即可。

Attention Here：由于编解码的原因，文件名和文件夹名不要为中文的，博客标题为中文的只需改.md文件里面的title为中文就可以了

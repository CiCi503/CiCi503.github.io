MySQL存储引擎简介

  • MySQL将数据以不同的技术存储在文件（内存）中，这种技术称为存储引擎
  • 每一种的存储引擎使用不同的存储机制，索引技巧，锁定水平，最终提供广泛且不同的功能
  • 在关系型数据库中，数据的存储是以表的形式实现的，所以存储引擎也可以称为表类型
  • MySQL指出的CNC引擎：MyISAM,InnoDB,Memory,CSV,Archive

子查询简介

  • 子查询(Subquery)是指出现在其他SQL语句内的SELECT子句
  • 子查询是指嵌套在查询内部，且必须始终出现在圆括号内
  • 子查询可以包含多个关键字或条件，如DISTINCT,GROUP BY,ORDER BY,LIMIT函数等
  • 子查询的外层查询可以是：SELECT,INSERT,UPDATE,SET,或DO
  • 这里的查询不是查找，而是SQL语句的统称，因为SQL就是结构化查询语言

MySQL(四)--约束

发表于 2017-02-11 | 分类于 MySQL | | 阅读量

约束

  • 约束的目的：保证数据的完整性和一致性
  • 约束分为表级约束和列级约束
    • 列级约束：约束只针对于一个字段
    • 表级约束：约束针对于两个或两个以上字段使用
  • 约束类型包括
    • 非空约束： NOT NULL
    • 主键约束：PRIMARY KEY
    • 唯一约束：UNIQUE KEY
    • 默认约束：DEFAULT
    • 外键约束：FOREIGN KEY，保存数据一致性，完整性，实现一对一或者一对多的关系

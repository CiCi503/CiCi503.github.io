MySQL(六)--子查询与连接

发表于 2017-02-14 | 分类于 MySQL | | 阅读量

子查询简介

  • 子查询(Subquery)是指出现在其他SQL语句内的SELECT子句
  • 子查询是指嵌套在查询内部，且必须始终出现在圆括号内
  • 子查询可以包含多个关键字或条件，如DISTINCT,GROUP BY,ORDER BY,LIMIT函数等
  • 子查询的外层查询可以是：SELECT,INSERT,UPDATE,SET,或DO
  • 这里的查询不是查找，而是SQL语句的统称，因为SQL就是结构化查询语言
  • 子查询可以返回标量、一行、一列或子查询
  • 子查询可以分为三种啊
    • 由比较运算符引发的子查询
    • 使用[NOT] IN 的子查询
    • 使用[NOT] EXISTS的子查询

使用[NOT] EXISTS的子查询

使用INSERT … SELECT插入记录

  • 由于记录的条目越来越多，会存在许多重复的信息，占用大量的空间，导致查询速度变慢，解决这种问题就用外键约束
  • INSERT … SELECT语句是将从一个数据表中查找出的特定记录批量插入到另一个数据表中

连接的语法结构

  • 基本结构就是：A表-连接类型-B表-两张表的连接条件
  • 连接类型分为三种
    • 内连接INNER JOIN
    • 左外连接LEFT OUTER JOIN
    • 右外连接RIGHT OUTER JOIN
  • 可以为表赋予别名
    • 数据表可以使用tbl_name alias_name或tbl_name alias_name赋予别名
    • table_subquery可以作为子查询使用在FROM子句中，这样的子查询必须为其赋予别名
  • 连接条件，使用ON关键字来设定连接条件，也可以使用WHERE来代替。通常使用ON关键字来设定连接条件，使用WHERE关键字进行结果集记录的过滤

    内连接 INNER JOIN

  • 显示左表及右表符合连接条件的记录，仅显示符合连接条件的记录
  • 即显示的内容啊是左表也有右表也有的，如果只有一个表有，另一个表没有，那是不会显示的
  • 如果使用内连接查找的记录在连接数据表中不存在，并且在WHERE子句中尝试以下操作：col_name IS NULL时，如果col_name被定义为NOT NULL，MySQL将在找到符合连接条件的记录后停止搜索更多的行。也就是两个表的限制条件冲突了，就停止搜索

外连接 OUTER JOIN

  • 左外连接：显示左表中的全部和右表中的符合连接条件的
  • 右外连接：显示右表中的全部和坐标中的符合连接条件的
  • 说明：以左外连接为例A LEFT JOIN B join_condition
    • 数据表B的结果集依赖数据表A，即数据表A有的记录在B表中显示出来，否则在B表中不能够显示
    • 数据表A的结果集根据左连接条件依赖所有的数据表（B表除外）
    • 左外连接条件决定如何检索数据表B(在没有指定WHERE条件的情况下)
    • 如果数据表A的某条记录符合WHERE条件，但是在数据表B不存在符合连接条件的记录，将生产一个所有列为空的额外的B行。
